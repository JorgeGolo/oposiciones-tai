Arquitectura de la Interfaz Frontend

Puntos clave sobre cómo se ha construido y estructurado la interfaz de usuario de este panel de estudio.

1. Angular & Angular Material 3

El proyecto utiliza Angular 18+ con componentes de tipo standalone y el sistema de diseño Material 3. Hemos seleccionado la paleta predefinida Azure/Blue durante la instalación, la cual inyecta los estilos globales e iconos de forma automática.

2. Sistema de Estilos Híbrido

Para evitar un diseño plano y mejorar el contraste visual, combinamos el potencial de ambos mundos:

  • Tokens Nativos: Mapeamos los colores directamente del inspector usando variables CSS globales como var(--mat-sys-primary) o var(--mat-sys-surface-container).
  • Clases Propias: Toda la estructura del Layout, la barra lateral (.sidebar) y la superior (.topbar) se controlan con clases SCSS personalizadas y encapsuladas.

3. Navegación SPA (Sidebar + Router)

El menú lateral utiliza <mat-nav-list> integrado con el enrutador nativo de Angular. Al usar routerLink en lugar de enlaces tradicionales, el contenido se intercambia al instante dentro del contenedor principal gracias a la directiva: 

<main class="content">
  <router-outlet></router-outlet>
</main>

La barra de rutas (app.routes.ts) gestiona la carga diferida (Lazy Loading) y la redirección automática al inicio: 

{ path: '', redirectTo: '/estructura', pathMatch: 'full' }

4. Modularidad con Angular CLI

Cada sección del panel se gestiona como un componente independiente y limpio generado directamente desde la terminal mediante la interfaz de comandos de Angular: 

ng generate component paginas/nombre-componente
