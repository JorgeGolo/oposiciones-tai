Conexión con el Backend (ASP.NET Core)

Configuración esencial para comunicar nuestro frontend de Angular con la Web API en desarrollo y producción.

1. Habilitar CORS en el Backend

Imprescindible para que el navegador no bloquee las peticiones. Añade esto en el Program.cs de tu API: 

builder.Services.AddCors(options =>
{
    options.AddPolicy("FrontendPolicy", policy =>
    {
        policy.WithOrigins(
                "http://localhost:4200",
                "https://api-oposicion-tai-jorgegl.azurewebsites.net"
              )
              .AllowAnyHeader()
              .AllowAnyMethod();
    });
});

2. Configurar Entornos en Angular

Genera los archivos de entorno ejecutando ng generate environments y configura tus URLs:

Desarrollo (environment.development.ts)

export const environment = {
  production: false,
  apiUrl: 'https://localhost:5026'
};

Producción (environment.ts)

export const environment = {
  production: true,
  apiUrl: 'https://api-oposicion-tai-jorgegl.azurewebsites.net'
};

Controlador del Backend (ASP.NET Core)

Rutas automáticas mediante atributos

En ASP.NET Core no existe un archivo de rutas centralizado como en otros frameworks. Las rutas se definen directamente en el controlador mediante atributos:

  • [ApiController]: marca la clase como controlador de API REST.
  • [Route("api/[controller]")]: genera el prefijo /api/Temas automáticamente a partir del nombre de la clase.
  • [HttpGet], [HttpPost], [HttpPut id], [HttpDelete( id )]: definen el método HTTP y el segmento de ruta de cada acción.

using Microsoft.AspNetCore.Mvc;
